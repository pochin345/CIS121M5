Imprecision of float

Trying to represent a number with a decimal point
as a binary number results in imprecise numbers.

For example. suppose I were try to multiply 1.3 by 1.3
In decimal that would result in 1.69
Not so as this simple exercise will show you.

Enter 1.3 at the prompt
the program multiplies the number by itself
you get 1.6900000000000002 when not rounded
1.69 when rounded

If you want to know why, click here

We're more interested in how to fix it:

round()

The round function:

  • fixes the imprecision of the float type
  • rounds the number to a specified number of decimal places
    typically 2, if you're working with currency

Task

Enter various numbers, and notice the difference
between not rounded and rounded numbers
when the number is multiplied by itself.

Number
(input)		 Not rounded Rounded to 2 decimals
1.3 1.6900000000000002 1.69
